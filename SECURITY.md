# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability in this project, please report it
responsibly through one of the following channels:

1. **GitHub Security Advisories (preferred):** Open a private advisory via
   [GitHub Security Advisories](../../security/advisories/new).
2. **Email:** Send details to the project maintainers at
   **security@hve-stem.dev**.

### What to Include

- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue or a proof-of-concept.
- Any suggested mitigations or fixes, if available.

## Response Timeline

| Stage                 | Target          |
|-----------------------|-----------------|
| Acknowledgement       | Within 48 hours |
| Initial assessment    | Within 7 days   |
| Fix or mitigation     | Within 30 days  |
| Public disclosure     | After fix ships  |

We will coordinate with reporters on disclosure timing and credit attribution.
